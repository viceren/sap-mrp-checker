## P0改进内容 (已实施)

### 1. 候选过滤: fixed_pos_morph的"0"答案直接丢弃
- **问题**: "0陷阱"——特征识别器把模糊区域误判为数字0, 导致大量0+0=0、0×N=0的错误答案
- **效果**: fixed_pos_morph准确率从7% → 不再参与0答案投票

### 2. 降权: fixed_pos_morph整体 ×0.3
- **原因**: 该方法准确率仅7%, 但通过策略一致性bonus获得虚高置信度
- **效果**: 降权后低于0.50阈值, 大部分被自然过滤

### 3. 降权: OCR原文字母占比>40%时 ×0.5
- **问题**: 如'8t1x'→9, '6t57'→11 等噪声污染
- **效果**: 含大量字母的OCR结果不再主导投票

### 4. 降权: "0"答案 ×0.6
- **原因**: 0答案准确率仅13% (0+N/0×N在真实验证码中较少)
- **效果**: 0答案不再通过高置信度压制其他候选

### 5. 策略一致性bonus收紧: 0.15/策略 → 0.08/策略, 上限0.40 → 0.16
- **问题**: 多个错误策略一致时, 会让错误答案获得虚高置信度
- **效果**: 减少假阳性高置信度

### 6. 拒识阈值: 0.2 → 0.50
- **原因**: 0.50以下候选准确率极低, 拒识后重试更优
- **效果**: 仅6%拒识率, 不影响有效尝试次数

### 典型错误案例分析

#### 错误类型1: "0陷阱" (fixed_pos_morph)
- #4: 实际 `1+4=5` → 识别为 `0×0=0` conf=0.85
- #5: 实际 `0+9=9` → 识别为 `0+0=0` conf=0.58
- #22: 实际(非0运算) → 识别为 `0×5=0` conf=0.80
- **根因**: 特征识别器对空白/模糊区域误判为数字0, 且0+N或0×N结果仍为0

#### 错误类型2: OCR噪声污染
- #1: OCR `'8t1x'` → 解析 `8+1=9`, 但实际不是这个算式
- #7: OCR `'631'` → 取数字6,3,1组合出9
- #8: OCR `'1aa'` → a→4, 得5
- **根因**: 字母被映射到数字/运算符后产生虚假算式

#### 错误类型3: 高置信度错误 (最危险!)
- #11: conf=0.97 ✗ `red_clean_ocr(loose):'6t57'→11`
- #15: conf=0.97 ✗ `raw_ocr:'3t5x'→8`
- #16: conf=0.97 ✗ `red_clean_ocr(medium):'8t5'→13`
- #20: conf=0.97 ✗ `red_clean_ocr(medium):'90x7'→0`
- #26: conf=0.97 ✗ `raw_ocr:'2t1'→3`

**高置信度不等于正确!** 0.97的置信度也有53%的错误率

## 问题根因分析

### 1. ddddOCR 对手写体算术式识别能力有限
- 单字符识别尚可, 但连续手写体(数字+运算符)混淆率高
- 运算符 `t`/`f` 被过度信任为 `+`
- 数字 `o`→0, `l`→1, `s`→5 等替换规则产生级联错误

### 2. 特征识别器(像素特征)精度不足
- 固定位置分割(x22-55, x58-98)可能偏移
- 对非标准字体/变形字符特征模板不够鲁棒
- 大量将非0字符误判为0

### 3. 投票机制放大了错误
- 多个策略给出不同错误答案时, 投票选出了"最一致"的错误
- 策略一致性加分(n_strategies bonus)让错误答案获得虚高置信度

### 4. 缺乏"拒绝"机制
- 即使所有策略都不确定, 也强制输出一个答案
- 应该在置信度低于阈值时返回None, 让重试逻辑处理

## 仍存在的问题

### 1. 0.50-0.65区间准确率仍低 (13%)
- 23个样本中仅3个正确
- 主要是red_clean_ocr(medium)的纯数字OCR结果(如'932','752')被误解析
- OCR识别出了数字但算式结构错误(如把"9+3"识别为"932")
- **需要P1级改进(专用模型)才能解决**

### 2. 高置信度假阳性仍然存在
- 0.97置信度的13个样本中仍有6个错误(46%错误率)
- 如'4x9'→36(×识别错误), '5x2'→10, '8t6'→14
- 这些是OCR本身识别错误, 无法通过降权规则过滤

## 后续改进方向

### P1 - 中期改进 (预计提升到60-70%)

#### E. 收集标注样本训练专用模型
- 已有50个样本(captcha_samples/), 需再收集150个
- 人工标注每张图片的正确答案
- 方案选择:
  - **方案A**: 微调ddddocr模型(需要GPU)
  - **方案B**: 训练轻量CNN分类器(ResNet-18, 分类100种算术结果0-81)
  - **方案C**: 两阶段检测+CRNN(先定位字符位置, 再分别识别)

#### F. 数据增强
- 对正确样本做旋转、缩放、加噪增强
- 用错误样本做困难负例训练

### P2 - 长期架构升级 (目标85%+)

#### G. 目标检测 + 字符识别
```
原图 → YOLO检测4个字符区域 → 分别CNN分类 → 组装算式 → 计算
```
优势: 避免固定位置假设, 处理任意布局

#### H. 端到端序列模型
```
原图 → CRNN/CNN+Transformer → 直接输出 "3+4=7"
```
优势: 利用上下文信息, 整体识别而非逐字
